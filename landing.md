**общее**
название - Snappit

продукт - декстоп утилита для macos (в будущем и windows с linux) которая продоставляет инструменты для работы с экраном - OCR (поддержка нативных языков системы с помощью macos vision и всех языков из tesseract), пипетка (HEX, RGB, HSL, OKLCH), экранная линейка, чтение QR кодов

Цель лендинга: прзентация продукта, дать пользователю ссылку для скачивания и покупки приложения

Лицензирование: Бесплатный trial на 14 дней или полная пожизненная лицензия за $7.99

**структура**

1. Первый экран (Hero Section)
   Это самый важный блок. Пользователь должен за 3 секунды понять: что это, для кого это и зачем это скачивать.

Заголовок (H1): Четкий и бьющий в цель.
Подзаголовок (H2): Расшифровка ценности.

Визуал (Hero Image/Video): Не используйте абстрактные иллюстрации. Лучше всего сработает короткое зацикленное видео (loop) или GIF, показывающее интерфейс в действии (как появляется панель инструментов поверх других окон).

CTA (Call to Action): Две кнопки.

Основная (яркая): «Скачать для macOS» (можно добавить версию ОС, например, macOS 12+).

Второстепенная (прозрачная/контурная): «Купить полную версию за $7.99».

Сноска: «Windows и Linux скоро» (ссылка на подписку на обновления).

2. Блок «Боли» (опционально, но эффективно)
   Короткий блок, объясняющий, почему текущий процесс работы неудобен.

Идея: «Перестаньте переключаться между пятью разными утилитами или гуглить онлайн-конвертеры».

3. Функционал (Features) — «Что внутри»
   Разбиваем продукт на конкретные инструменты. Для ЦА важна техническая точность. Лучше всего сделать сетку 2x2.

Инструмент 1: OCR (Распознавание текста)

Иконка: copy.svg

Текст: «Копируйте любой текст, который видите на экране. Работает с видео, PDF, изображениями и защищенными сайтами. Поддержка болл 100 языков».

Инструмент 2: Пипетка (Color Picker)

Иконка: color-dropper.svg.

Текст: «Определяйте цвет любого пикселя. Мгновенное копирование в HEX, RGB, HSL, OKLCH. Идеально для фронтенда и UI-дизайна».

Инструмент 3: Экранная линейка (Ruler)

Иконка: ruler.svg

Текст: «Измеряйте отступы, высоту и ширину элементов с точностью до пикселя. Проверяйте верстку за секунды».

Инструмент 4: QR Сканер

Иконка: qr.svg

Текст: «Читайте QR-коды прямо с рабочего стола без использования телефона».

4. Блок «Сценарии использования» (Use Cases)
   Покажите, как продукт встраивается в жизнь разных сегментов ЦА. Можно оформить в виде табов или слайдера.

Для Разработчиков: «Быстро скопировать текст ошибки из лога, который нельзя выделить, или узнать HEX-код кнопки на макете».

Для Дизайнеров: «Проверить pixel-perfect верстку и собрать палитру цветов с референсов».

Для Студентов/Исследователей: «Вытягивать цитаты из сканов книг, слайдов лекций и видеоуроков».

5. Roadmap
   Укажи будущий roadmap для продукта

- автоматизация после захвата текста (например перевод или спросить AI)
- релиз на windows/linux

6. Подвал (Footer)
   Ссылки: Скачать, База знаний (Docs), Политика конфиденциальности (очень важно для App Store и доверия), Условия использования.

Контакты: Email поддержки, Twitter/X разработчика.

**технические требования**

- Стиль - Используйте "Bento Grids" (блочная структура, как у Apple)
- Типографика - Используйте системные шрифты (San Francisco для Mac) или качественные гротески (Inter, Roboto)
- Стилизация: tailwind
- Весь текст должен быть на английском языке.
- Для всех видео/картинок создай placeholder без самого контента (контент будет добавлен позже)
- Используй цвета из global.css
- Сайт должен поддерживать Dark Mode и светлую тему.
- Сайт должен быть адаптивным и работать на всех устройствах (мобильные, планшеты, десктопы).
- В качестве иконок инструментов используй иконки из папки public (copy.svg, color-dropper.svg, ruler.svg, qr.svg)
- В качестве лого используй иконку из папки public (logo.png/logo-128.png)
